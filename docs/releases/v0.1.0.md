# RanD v0.1.0

## 概要

RanD の初回リリースです。R&D Agent の導入層と実行層を束ねる母艦 repo として、installer、research runtime、Kestra サンプル flow、要件定義から検収までの文書群を公開します。

## 含まれる内容

- `r-and-d-agent-installer` による pinned install と local/remote 解決
- `research-runtime` による preset 実行、heartbeat、state-aware な収集と artifact 保存
- `kestra/flows` による manual、daily、nightly、heartbeat の公開サンプル flow
- `README`, `architecture`, `requirements`, `specification`, `evaluation` の整備

## 今回の検収反映

- `insight-agent` / `experiment-gate` の内部 status を RanD の top-level status へ伝播
- `memx_journal.json` と `tracker_sync.json` の run artifact schema を root + entry/event 形式へ統一
- 既存 state log の `schema_version` 補完
- README に残留リスクを追記

## 確認済み

- `python -m unittest discover tests`
- `heartbeat --dry-run`
- `env-check`
- live `run-once` による `ok`, `failed`, `degraded` 系の確認

## 残留リスク

- live fetch と live LLM 実行は外部依存に左右される
- notification / replay / dedupe の実件数は `pulse-kestra` 側の flow output と taskstate 記録に依存する
- peer repo 側の API / schema 変更は定期的な統合確認が必要
